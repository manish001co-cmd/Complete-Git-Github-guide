              Complete Git and GitHub Guide

              Git

              A Version Control System (VCS) is a tool that helps track changes in code over time.

              Git is a distributed Version Control System. It is:

              • Popular
              • Free and Open Source
              • Fast and Scalable

              Git is mainly used to:

              • Track the history of changes in a project
              • Collaborate with other developers
              • Manage different versions of code
              • Backup source code safely

              Download and Install Git for Windows

              1. Search for "Git Download" or use the official website: Click here to download Git for Windows
              2. Open the installer and click Next.
              3. Continue with the default installation settings.
              4. Configure the installation options.
              5. Proceed with the remaining setup steps.
              6. Choose the default editor used by Git.
              7. Adjust the name of the initial branch in new repositories.
              8. Adjust your PATH environment variables.
              9. Choose the SSH executable.
              10. Choose the HTTPS transport backend option.
              11. Configure the line ending conversions.
              12. Configure the terminal emulator to use with Git Bash.
              13. Choose the default behavior of git pull.
              14. Choose a credential helper to store your logins.
              15. Complete the installation.

              GitHub

              GitHub is a web-based platform that allows developers to store, manage, and share their code using Git.

              Website: https://github.com

              What is GitHub?

              GitHub is often called the social network for developers. It allows programmers to share projects, collaborate with other developers, and contribute to open-source software.

              Folder → Repository

              Create a GitHub Account

              1. Visit the GitHub website: GitHub
              2. Click on Sign Up to create a new account. If you already have an account, click Sign In.
              3. Use your email address to create an account.
              4. After successfully logging in, you will see your dashboard.
              5. Click Create Repository to create a repository and upload your files.
              6. Commit your files after uploading them.

              Commit: A saved snapshot of your project changes.

              Commit Message: A short description explaining what changes were made.

              Configuring Git

              After installing Git, we need to configure it.

              Configuring Git means telling Git who you are, so that every commit you make is linked to your name and email.

              Git stores this information in your commits like this:

              Author: John Doe <john@example.com>

              You usually configure Git only once after installing it.

              1. Check if Git is Installed

              Open Command Prompt or VS Code Terminal and run: 

              git --version

              Example output:

              git version 2.50.1

              2. Configure Your Name

              Use the following command:

              git config --global user.name "Your Name"

              Example:

              git config --global user.name "Manish Kumar"

              3. Configure Your Email

              Use the same email address that you use for GitHub. 

              git config --global user.email "your_email@example.com"

              4. Verify Your Configuration

              git config --global --list

              Example output:

              user.name=Manish Kumar
user.email=abc@gmail.com

              Basic Git Commands

              Git Init

              What is git init?

              The git init command is used to initialize a new Git repository in an existing project folder.

              When you run this command, Git starts tracking changes in the project by creating a hidden .git directory.

              Syntax

              git init

              Example

              git init

              What happens after using git init?

              • A hidden .git folder is created inside the project directory.
              • Git starts tracking changes in the project.
              • The folder becomes a local Git repository.
              • You can now use Git commands such as git add, git commit, and git status.

              Use Cases

              • Start version control for a new project.
              • Convert an existing folder into a Git repository.
              • Begin tracking changes in source code.

              Example Workflow

              mkdir my-project
cd my-project
git init

              After running the git init command, the folder becomes a Git repository.

              Verify Initialization

              Run the following command:

              git status

              Example output:

              On branch main

No commits yet

nothing to commit
(create/copy files and use "git add" to track)

              Important Note

              The git init command creates only a local repository on your computer. To upload the project to GitHub, you need to connect it to a remote repository and use git push.

              Example Complete Workflow

              git init
git add .
git commit -m "Initial commit"
git remote add origin <repository-url>
git push -u origin main

              Importance of git init

              The git init command is the first step in using Git. It enables version control, helps track project history, and allows collaboration through GitHub.

              Git Clone

              What is git clone?

              The git clone command is used to create a copy of an existing remote repository from GitHub (or another Git server) to your local computer.

              It downloads all the files, branches, and commit history from the remote repository.

              Syntax

              git clone <repository-url>

              Example

              git clone https://github.com/username/project-name.git

              What happens after cloning?

              • A new folder with the repository name is created.
              • All project files are downloaded.
              • The complete commit history is copied.
              • The remote repository is automatically connected as origin.

              Use Cases

              • Download an existing project from GitHub.
              • Contribute to open-source projects.
              • Work on a project locally.

              Example Workflow

              git clone https://github.com/username/project.git
cd project

              After entering the project folder, you can start making changes.

              Git Status

              What is git status?

              The git status command shows the current state of your Git repository.

              It tells you:

              • Which files have been modified.
              • Which files are staged for commit.
              • Which files are untracked.
              • Which branch you are currently working on.

              Syntax

              git status

              Example Output

              On branch main

Changes not staged for commit:
  modified: index.html

Untracked files:
  style.css

              Meaning of the Output

              • Modified files → Files changed but not staged.
              • Untracked files → New files not added to Git.
              • Staged files → Files ready to be committed.

              Common Workflow

              git status
git add .
git status
git commit -m "Updated project files"

              Importance of git status

              The git status command helps developers track changes before committing code, reducing mistakes and improving project management.

              Git Add

              What is git add?

              The git add command is used to move changes from the Working Directory to the Staging Area.

              Before creating a commit, Git requires you to select which files should be included in the next commit. This process is called staging.

              Syntax

              git add <file-name>

              Example

              git add index.html

              The above command stages only the index.html file.

              Add All Files

              git add .

              What happens after using git add?

              • Modified files are moved to the staging area.
              • Git prepares the selected files for commit.
              • Changes are not yet permanently saved.

              Use Cases

              • Stage a specific file.
              • Stage multiple files.
              • Stage all project files before committing.

              Example Workflow

              git status
git add .
git status

              After running git add, the staged files are ready to be committed.

              Git Commit

              What is git commit?

              The git commit command is used to save the staged changes permanently in the local Git repository.

              A commit acts like a snapshot of your project at a specific point in time.

              Syntax

              git commit -m "commit message"

              Example

              git commit -m "Added login page"

              The commit message should clearly describe the changes made.

              What happens after committing?

              • A new commit is created.
              • Changes are saved in Git history.
              • Each commit has a unique commit ID (hash).
              • You can return to previous versions if needed.

              Best Practices for Commit Messages

              • Use clear and meaningful messages.
              git commit -m "Fixed navbar alignment"
git commit -m "Added CSS animations"
git commit -m "Updated README file"

              Example Workflow

              git add .
git commit -m "Initial project setup"

              Importance of git commit

              The git commit command helps developers track project history, collaborate efficiently, and manage different versions of a project.

              Git Branches

              What is a Git Branch?

              A branch in Git is an independent line of development that allows developers to work on new features, bug fixes, or experiments without affecting the main project.

              Branches help multiple developers work on the same project simultaneously without interfering with each other's work.

              By default, Git creates a branch named main (or master in older versions).

              Why Use Branches?

              • Allow developers to work on new features safely.
              • Prevent changes from affecting the main project.
              • Enable multiple developers to work simultaneously.
              • Make it easy to test experimental code.

              View Existing Branches

              git branch

              Example Output:

              * main
  login-feature
  bug-fix

              The * symbol indicates the currently active branch.

              Create a New Branch

              git branch <branch-name>

              Example:

              git branch login-page

              This creates a new branch named login-page.

              Switch Between Branches

              git checkout <branch-name>

              Example:

              git checkout login-page

              Git now changes the working directory to the selected branch.

              Create and Switch to a Branch

              git checkout -b <branch-name>

              Example:

              git checkout -b dashboard

              This command creates the dashboard branch and switches to it immediately.

              Modern Alternative

              git switch <branch-name>

              Example:

              git switch login-page

              To create and switch simultaneously:

              git switch -c dashboard

              Delete a Branch

              git branch -d <branch-name>

              Example:

              git branch -d login-page

              Rename a Branch

              git branch -m <new-branch-name>

              Example:

              git branch -m feature-login

              Merge Branches

              git checkout main
git merge login-page

              This merges the login-page branch into main.

              Example Workflow

              git branch login-page
git checkout login-page

git merge login-page

              Importance of Git Branches

              • Enables parallel development.
              • Keeps the main branch stable.
              • Simplifies collaboration.
              • Helps manage features and bug fixes efficiently.

              Git branches are one of the most powerful features of Git and are widely used in professional software development.

              Git Push

              What is git push?

              The git push command is used to upload local repository changes to a remote repository such as GitHub.

              After making commits in your local repository, you use git push to send those changes to GitHub so that other developers can access them.

              Syntax

              git push <remote-name> <branch-name>

              Example

              git push origin main
              • origin → Name of the remote repository.
              • main → Name of the branch.

              First Time Push

              When pushing a branch for the first time, use: 

              git push -u origin main

              The -u option sets the upstream branch, allowing future pushes using only: 

              git push

              What happens after pushing?

              • Local commits are uploaded to GitHub.
              • The remote repository is updated.
              • Other developers can access the latest changes.

              Use Cases

              • Upload project changes to GitHub.
              • Share code with team members.
              • Create backups of your project online.

              Example Workflow

              git add .
git commit -m "Added login page"
git push origin main

              Importance of git push

              The git push command helps synchronize local work with the remote repository, enabling collaboration and secure code storage.

              Git Pull

              What is git pull?

              The git pull command is used to download the latest changes from a remote repository and merge them into your local repository.

              It keeps your local project up to date with changes made by other developers.

              Syntax

              git pull <remote-name> <branch-name>

              Example

              git pull origin main

              What happens after pulling?

              • Latest commits are downloaded from GitHub.
              • Changes are merged into the current branch.
              • Your local repository becomes up to date.

              Use Cases

              • Get updates from team members.
              • Synchronize your project with GitHub.
              • Download newly added files and changes.

              Example Workflow

              git pull origin main
git status

              Best Practice

              Always run git pull before starting work to ensure you have the latest version of the project.

              Importance of git pull

              The git pull command helps prevent conflicts and keeps all developers working with the latest version of the code.

              Complete Git Workflow

              What is Git Workflow?

              A Git workflow is a sequence of Git commands that developers use to manage, track, and share changes in a project.

              It helps developers collaborate efficiently and maintain different versions of the project.

              Basic Git Workflow

              The following steps represent a typical Git workflow used in software development: 

              git init
git status
git add .
git commit -m "Your commit message"
git remote add origin <repository-url>
git push -u origin main
git pull origin main

              Step-by-Step Explanation

              1. Initialize a Repository

              git init

              Creates a new local Git repository in the current folder.

              2. Check Repository Status

              git status

              Displays the current state of the repository, including modified, staged, and untracked files.

              3. Stage Changes

              git add .

              Adds all modified and new files to the staging area.

              4. Create a Commit

              git commit -m "Your commit message"

              Saves the staged changes permanently in the local repository.

              Example:

              git commit -m "Added login page"

              5. Connect to GitHub

              git remote add origin <repository-url>

              Links the local repository to a remote GitHub repository.

              Example:

              git remote add origin https://github.com/username/project.git

              6. Push Changes to GitHub

              git push -u origin main

              Uploads local commits to the remote GitHub repository.

              The -u option sets the upstream branch for future pushes.

              7. Pull Latest Changes

              git pull origin main

              Downloads and merges the latest changes from GitHub into the local repository.

              Importance of Git Workflow

              • Helps track project history.
              • Enables collaboration among developers.
              • Prevents data loss.
              • Maintains different versions of the project.
              • Simplifies project management.

              Real-Life Example

              1. Initialize Git using git init.
              2. Create or modify files.
              3. Check changes using git status.
              4. Stage files using git add ..
              5. Save changes using git commit.
              6. Upload the project to GitHub using git push.
              7. Download updates using git pull.

              Following this workflow ensures that your project remains organized and synchronized with GitHub.
